Content type

An HTTP request's "Content-Type" header is used to specify the media type of the resource that is contained in the message body. It describes the kind of data as well as its subtype when appropriate. To correctly interpret the data being sent, the content type is necessary for both the client and the server.

Why do we need it?

  • Error handling: the server can provide an informative error when getting a request with an unexpected content type.
  • Proper handling: knowing the data type helps the server in handling the request.
